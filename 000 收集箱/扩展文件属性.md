# 简介

扩展文件属性（Extended Attributes）是文件系统中存储的附加元数据键值对。它与文件内容分离，允许操作系统或应用程序为文件自定义信息而无需修改文件本体。每个属性采用 `命名空间.名称=值` 的格式。

Linux 系统通过 `attr` 命令或更推荐的 `getfattr` / `setfattr` 命令操作扩展属性；macOS 则统一使用 `xattr` 命令。

---

# 正文

## 定义与结构

扩展文件属性是与文件或目录永久关联的 `名称:值` 对，提供传统 Unix 权限之外的元数据存储。

## 命名空间

扩展文件属性使用多个互不相交的命名空间来隔离不同用途的属性：

| 命名空间   | 用途                 | 访问权限                                 |
| ---------- | -------------------- | ---------------------------------------- |
| `user`     | 用户自定义的任意数据 | 受文件普通权限保护，文件所有者可控制读写 |
| `security` | SELinux 安全标签     | 仅超级用户可访问（需显式指定选项）       |
| `system`   | 系统功能（如 ACL）   | 内核使用                                 |
| `trusted`  | 特权应用程序使用     | 仅超级用户可访问                         |

## 常见用途

- 存储文件元信息：如文档编码、缩略图、作者、版本号；
- 安全标签：SELinux 的 `security.selinux` 属性，macOS 的 `com.apple.quarantine` 属性记录下载来源；
- 校验：如文件哈希值或应用程序特定数据。

## 常用工具与参数选项

### `attr`

`attr` 命令是 Linux 最初为 XFS 文件系统设计，现可用于任何支持扩展文件属性的文件系统，但官方更推荐使用 `getfattr` 和 `setfattr` 命令代替。

```bash
attr -s <属性名> -V <值> <文件> # 设置属性
attr -g <属性名> <文件> # 获取属性值
attr -l <文件> # 列出所有属性
attr -r <属性名> <文件> # 删除属性
```

### `getfattr` / `setfattr`

这是 Linux 下更通用的命令。

```bash
getfattr -d <文件> # 列出所有属性及值
getfattr -n <属性名> <文件> # 获取指定属性值

setfattr -n <属性名> -v <值> <文件> # 设置属性
setfattr -x <属性名> <文件> # 删除属性
```

### `xattr`

macOS 使用统一的 `xattr` 命令，不支持命名空间前缀。

```bash
xattr <文件> # 仅列出属性名
xattr -l <文件> # 列出属性名及对应值
xattr -p <属性名> <文件> # 打印指定属性的值

xattr -w <属性名> <值> <文件> # 写入或覆盖属性
xattr -d <属性名> <文件> # 删除指定属性
xattr -c <文件> # 清除文件所有扩展属性
xattr -cr <目录> # 递归清除目录下所有文件的扩展属性
```

常用命令：

```bash
# 网络下载文件会被标记为 `com.apple.quarantine`，尝试打开时会触发警告
xattr -d com.apple.quarantine <文件>
```
